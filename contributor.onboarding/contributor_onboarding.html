    Working Agreements

    Minimum rules that prevent chaos.

    Small = professional

    1) Small is Professional

    One task = one PR = one deliverable. No stealth refactors.

    • Keep changes scoped.
    • Ship a clean win, then grab the next win.
    • If you found “extra fixes,” log them as separate tasks.

    2) We Don’t Build on Vibes

    If a task is missing clarity, it’s discussion—not implementation.

    Goal Scope Out-of-scope Acceptance Done

    3) Decisions Have Owners

    Founder owns product direction + final calls. You own implementation within scope.

    • Want to change scope? Propose it—don’t stealth it.
    • Subjective UI? Founder decides quickly; we move on.

    4) Document As You Go

    No essays. Just the essentials.

    • What changed
    • Why
    • How to run/test
    • Gotchas

    The Honest Helper

    Confidence with humility. Ask early. No guessing in risky zones.

    Definition of Done

    “Works in preview” is the baseline. Add test notes + tiny docs note.

    No security regressionsNo secrets, no unsafe inputs, no sketchy defaults.
